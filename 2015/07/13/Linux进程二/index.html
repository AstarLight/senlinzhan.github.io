发表于 2015-07-13   |   分类于 系统编程

清除子进程

　　当子进程终止之后，父进程应当清除子进程，否则子进程将成为僵尸 ( Zombies ) 进程。僵尸进程会占用系统资源，因此，long-running 程序应当清除已经终止的子进程。另一方面，当父进程终止时，它的子进程还未被清除，那么，内核将调用 init 进程来清除这些僵尸进程。

　　在父进程中可以使用waitpid()来等待子进程终止，并清除子进程。我们可以使用pid来指定需要等待的子进程，waitpid()会暂停执行当前的进程，以等待这个子进程的终止。要是pid的值是-1，那么，当前进程将会等待它的子进程中其中一个进程的终止。waitpid()将会返回终止的子进程的 ID。

　　要是进程不存在子进程，那么调用waitpid()则会返回-1并将errno设置成ECHILD，另一方面，要是waitpid()被信号中断，那么，waitpid()则会返回-1并将errno设置成EINTR。

程序的载入与执行

　　通常使用execve()来加载并执行filename指定的可执行文件，正常情况下execve()并不会返回，若遇到错误，则会返回-1。实际上，执行execve()的时候并不会创建新的进程，即进程的 ID 保持不变。进程的地址空间会被覆写，但新程序会继承进程原来打开的所有文件描述符。
