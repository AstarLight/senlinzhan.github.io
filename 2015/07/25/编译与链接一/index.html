编译与链接（一）-- 编译过程

发表于 2015-07-25   |   分类于 体系结构

编译过程

　　一般来说，将源文件编程成可执行文件需要经过这几个阶段：

  • 预处理器对源文件进行预处理。
  • 再经过编译器编译成汇编源代码。
  • 汇编器将汇编代码转换成目标文件。
  • 链接器将多个目标文件以及库文件链接成可执行文件。

　　编译过程的第一步是执行预处理，预处理包括三部分：头文件包含、宏替代以及条件编译。

编译器做了什么

　　编译器和汇编器负责生成可重定位 ( Relocatable ) 目标文件，其格式通常为 ELF 格式：

  • .text : 程序编译而成的机器码。
  • .data : 初始化的全局变量。
  • .bss : 未初始化的全局变量，在目标文件中，.bss 区段实际上并不占据任何磁盘空间。只有当程序被加载到内存时，系统才会为 .bss 区段分配空间。
  • .symtab : 符号表，包含函数定义及声明的信息，全局变量定义及声明的信息，但不包含局部变量的信息。

　　当 C 编译器遇到未定义的全局变量以及未定义的函数时，则会假定它们的定义存在于其它 C 文件：

  • U 表示未定义的引用。
  • t 表示函数定义在文件作用域，T 表示函数定义在全局作用域。
  • d 表示初始化的全局变量 ( 文件作用域 )，D 表示初始化的全局变量 ( 全局作用域 )。
  • b 表示未初始化的全局变量 ( 文件作用域 ），B 或 C 表示未初始化的全局变量 ( 全局作用域 )。
