Linux 网络编程（五）-- I/O Multiplexing

发表于 2015-07-20   |   分类于 网络编程

I/O 多路复用

　　我们知道，在多线程的 TCP Server 里面，主线程使用accept()负责监听 Client 的连接请求，当主线程接受 Client 的连接之后，主线程将创建一个 peer 线程来负责响应 Client。

　　如果我们要求用户能通过在终端输入命令与这个 Server 进行交互，那么 Server 需要同时处理两种 I/O 事件，一个来自对 Client 连接的监听，另一来自响应用户在终端输入的命令。但是由于accept()是阻塞的，也就是说 Server 无法同时响应这两种 I/O 事件。

　　I/O 多路复用使得我们可以同时监视多个文件描述符。

select()

　　假定三个集合中最大的文件描述符的值是maxfd，那么numfds的值应当设定为maxfd + 1。

　　通过select()同时监视多个文件描述符集合，这样就可以知道哪些文件描述符 ready for reading，哪些文件描述符 ready for writing，还有哪个文件描述符 raised exception 了。参数timeout用来指定select()阻塞的时间上限。要是没有错误发生，那么，select()将会返回处于 ready state 的文件描述符的总数。

　　例如，我们可以通过FD_SET()将多个 socket 描述符添加进readfds，这样调用select()时，select()就会阻塞，要是readfds中存在 socket 是可读的 ( 有数据 )，那么select()将会返回，接着就可以通过FD_ISSET()来判断哪个 socket 是可读的了。

　　fd_set实际上是结构体，在调用select()之前，需要先使用FD_ZERO()和FD_SET()来初始化readfds、writefds和execeptfds指向的结构体。另一方面，需要注意select()将会修改这些结构体，这就是说当select()返回时，这些结构体里面仅仅包含处于 ready state 的文件描述符。若果我们需要在循环体之中调用select()，那么需要保证在调用select()之前重新初始化这些结构体。
