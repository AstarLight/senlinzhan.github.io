线程 ID

　　我们知道，操作系统为每个进程都赋予一个唯一的 ID，但与此不同，由于线程是存在于进程中的，因此，在同一进程里面，线程的 ID 都是唯一的。
　　另一方面，线程 ID 的类型是pthread_t，根据系统的不同，pthread_t可能是一个struct也可能是整数，因此，我们不能预先假定pthread_t的实际类型。

创建线程

　　我们可以使用pthread_create()来创建线程，其函数原型：

　　pthread_create()接受 4 个参数：

  • tidp指向一个pthread_t类型的变量，当pthread_create()成功返回时，tidp指向的内存将被设置成线程的 ID。
  • attr则表示自定义的线程属性，当它的值是NULL时，则表示线程使用默认的属性。
  • start_rtn表示线程将要执行的函数。
  • arg则表示start_rtn的参数。

　　通常，pthread_create()将返回 0，但当pthread_create()无法创建线程时，会将 error code 作为返回值返回。

分离线程

　　在任一时刻，线程是 joinable 或者是 detached 的。我们知道，pthread_join()用来等待特定的 joinable 线程终止，并清除这个线程。那么，detached 的线程在终止时，操作系统将自动释放这个线程的资源。
　　线程在创建之时就是 joinable 的，但可以调用pthread_detach()转变成 detached 的：

　　通过传递pthread_self()的返回值给pthread_detach()，线程可以 detach 其自身。在一些应用程序，例如 Web Server 中，子线程负责与特定的 client 进行交互，这时，这时主线程并不想等待子线程终止，那么子线程可以 detach 其自身，当子线程终止时，其资源将由操作系统回收。

线程终止

　　若任何线程调用了exit()，_Exit()或_exit()，那么，进程中的所有线程都会终止，当然，在main()中调用return语句也会终止所有的线程。
　
　　单个线程有三种方式来退出其自身，而不影响其它的线程：

  • 线程的 start routine 正常返回。
  • 线程可以被同一进程中的其它线程取消。
  • 线程调用pthread_exit()退出。
