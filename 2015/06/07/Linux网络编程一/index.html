Linux 网络编程（二）-- 基本概念

发表于 2015-06-07   |   分类于 网络编程

Socket

　　不同的应用程序可以使用 socket 进行彼此间的通信。根据使用目的的不同，socket 有两种类型：

  • Unix domain socket 允许同一主机的应用程序相互通信。
  • Internet domain socket 则允许不同主机的应用程序相互通信。

　　显然，我们需要使用 Internet domain socket 来支持不同应用程序的网络通信。在网络编程里面，常见的是 Client-Server 模式，也就是一个服务端响应多个客户端的模式。为使得 Client 和 Server 之间进行通信，Client 和 Server 各自都要创建一个 socket，并且利用 socket 进行通信。

　　那么，要怎样才能创建 socket 呢？可以使用socket()系统调用：

　　我们可以通过socket()的参数来指定特定的 socket 类型：

  • domain可以是AF_UNIX、AF_INET或AF_INET6。AF_UNIX表示前面提到的 Unix domain socket，AF_INET表示使用 IPv4 协议的 Internet domain socket，而AF_INET6则表示使用 IPv6 协议的 Internet domain socket。
  • type的值可以是SOCK_STREAM或SOCK_DGRAM，其中SOCK_STREAM表示 TCP socket，而SOCK_DGRAM则表示 UDP socket。
  • protocol的值通常指定为 0。

IP 地址

　　Internet domain socket 有两种类型的 IP 地址，IPv4 以及 IPv6 这两种。实际上大多数与 socket 相关的系统调用能够同时处理 IPv4 的地址和 IPv6 的地址，例如bind()系统调用：

　　让我们看看 Linux 中的sockaddr结构体：

　　可以将sockaddr看成地址模板，因为我们并不会直接使用sockaddr结构体。实际上我们使用sockaddr_in来表示 IPv4 的地址，而使用sockaddr_in6来表示 IPv6 的地址。
　　在实际使用的时候，例如，假定我们需要将一个sockaddr_in *类型的变量传递给bind()，那么，我们可以先将其显式地转换成sockaddr *类型，再传递给bind()。同样道理，也可以将sockaddr_in6 *类型的变量显式转换成为sockaddr *类型。通过这样做，bind()就能处理 IPv4 的地址，也能处理 IPv6 的地址了。

　　host参数可以是域名，例如"www.google.com"，也可以是 IPv4 或是 IPv6 的字符串地址。而service实际上是端口号，可以显示的指定端口号 ( 例如"80" )，也可以使用 "http"、"tftp"或"smtp"等。

　　使用getaddrinfo()进行 DNS 查询时，可能会有较多的符合情况的查询结果，因此，getaddrinfo()会通过参数res来返回一个链表，链表包含多个addrinfo结构体，每个结构体都包含相应的地址信息。我们可以将res传递给freeaddrinfo()用来释放链表的内存。

　　你可能会奇怪，参数hint的作用是什么？实际上，hint作为一个模板，用来过滤getaddrinfo()的查询结果，也就是说hint将指导getaddrinfo()如何执行查询工作。

　　在将hint传递给getaddrinfo()之前，需要使用memset()将hint的成员均设置为 0，接着，可以填充hint的ai_family、ai_flags、ai_protocol和ai_socktype这四个成员，其它的成员均设置成 0。
