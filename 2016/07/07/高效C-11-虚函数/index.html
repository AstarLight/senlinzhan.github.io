善用 override

　　派生类可以覆写 ( Override ) 基类的成员函数，但若要实现正确地覆写，则需要满足这些要求：

  • 函数的名字保持一致（除了析构函数）。
  • 函数的参数类型保持一致。
  • 函数的const性保持一致。
  • 函数的返回值类型、exception specifications 要保持一致。

　　但在 C++11 中，还需要满足函数的引用修饰 ( reference qualifiers ) 保持一致。

　　大多数情况下，在派生类中覆写基类的函数，我们需要满足上面的五点要求，C++11 中提供了override关键字，用来在编译期间检测派生类是否正确地覆写了基类的函数。

构造、析构

　　虚函数的使用法则是：绝不在构造函数和析构函数中调用虚函数。

　　我们知道，在派生类对象初始化之前，基类的数据成员会先被初始化。我们在Base::Base()中调用play()，这时由于派生类的数据成员还没有被初始化，因此调用的Base::play()而不是Derived::play()。

　　同样，我们知道，在派生类对象析构时，派生类的数据成员先被析构，接着再析构基类的数据成员，同样说明了在析构函数中不能调用虚函数。
