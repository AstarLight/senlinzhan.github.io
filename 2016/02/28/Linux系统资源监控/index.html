Linux 系统：资源管理

发表于 2016-02-28   |   分类于 系统管理

进程监控

　　top 命令能动态地显示系统的状态，默认情况下每 3 秒显示一次。top 会根据当前进程所占用的 CPU 时间，按顺序显示进程的信息。

　　22:13:35 表示当前时间，6:27 表示计算机从启动至今已运行 6 小时 27 分。6 users 表示 6 个登录用户。load average 表示系统平均负载，三个数值分别表示前 1 分钟、前 5 分钟和前 15 分钟的系统平均负载。在单个 CPU 的机器上，load average 为 1.0 则表示系统满载。在 n 个 CPU 的机器上，load average 为 n 则表示系统满载。

　　Tasks 则表示进程的数目以及进程的状态。%Cpu(s) 表示 CPU 执行的进程的特性。6.4 us 表示 6.4% 的 CPU 用于执行用户进程，2.3 sy 表示 2.3% 的 CPU 用于执行系统进程，0.0 ni 表示 0.0% 的 CPU 用于执行由 nice 命令指定的进程，91.0 id 表示 91.0% 的 CPU 是空闲的，0.3 wa 表示 0.3% 的 CPU 用于等待 I/O。

　　Mem 表示物理内存的使用情况，Swap 表示虚拟内存的使用情况。

　　在运行 top 时，我们可以通过键盘向 top 发送命令：空格：立即更新显示内容，M：按当前内存使用量排序，T：按照 CPU 累积使用量排序，P：按照当前 CPU 使用量排序（默认情况），u：仅显示指定用户的进程，f：显示不同的统计信息来显示，?：显示 top 命令的用法。
