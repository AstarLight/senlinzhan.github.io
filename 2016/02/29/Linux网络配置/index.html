Linux 系统：网络配置

发表于 2016-02-29   |   分类于 系统管理

子网

　　一个主机可以用多个 IP 地址，可以运行 ifconfig 来查看主机地址：

　　可以看到，本机的 IPv4 地址为 172.18.141.118，子网掩码为 255.255.255.0。如果一组互相连接的主机，并且它们的 IP 地址按一定次序排序，那么它们就组成一个子网。例如，172.18.141.1 到 172.18.141.254 就可以组成一个子网，这个子网的网络前缀（network prefix）就是 172.18.141.0，子网掩码是 255.255.255.0，这个子网可记为 172.18.141.0/255.255.255.0。

　　子网还可以使用 CIRD（Classless Inter-Domain Routing）表示法，例如，172.18.141.0/255.255.255.0 可以表示成 172.18.141.0/24。255.255.255.0 的二进制形式为 24 个 1 和 8 个 0，因此可以使用 24 来代表 255.255.255.0。另一个例子是 255.255.255.192 可以表示成 26。

　　假定我们知道子网 172.18.141.0/24，怎样判定这个子网包含哪些主机呢？172.18.141.0 的二进制表示为 10101100 00010010 10001101 00000000，我们将最后 8 位的任何位设置成 1（除了 00000000 和 11111111），就可以得到子网中的 IP 地址。

路由

　　假定主机 10.23.2.4 在局域网 10.23.2.0/24 中，那么它可以直接访问这个局域网中的任何主机。但是要是它想访问这个局域网之外的主机，那么就必须通过 IP 为 10.23.2.1 的路由器。

　　可以使用 route -n 来显示路由表：

　　“目标”就是网络前缀的意思，这里有两个网络 0.0.0.0/0 和 172.18.141.0/24。“标志”中的 U 表示路由器是活动的。0.0.0.0/0 匹配任何局域网之外的主机，“标志”中的 G 表示需要通过网关 172.18.141.254 才能访问这个网络。而 172.18.141.0/24 的“标志”没有 G，表示可以直接访问这个网络中的主机。

　　0.0.0.0/0 是默认路由，并且其网关为默认网关。当我们访问的主机不再局域网之内，就需要通过默认网关去访问。
